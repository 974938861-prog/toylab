# ToyLab Blocks 规则设计

> 与 [Blockly 官方文档](https://developers.google.com/blockly/guides/get-started/toolbox?hl=zh-cn) 保持一致：工具箱用 JSON 定义、块可拖入**编程工作区**；事件/执行/控制/运算分别对应 Blockly 的**帽子块、堆叠块、C型块、数据块**。

---

## 一、设计原则

1. **画布驱动**：编程区左侧仅显示与**当前仿真画布上的电子模块**相关的 blocks，避免无关块干扰。
2. **分类与块形态**：blocks 分为四类，与 Blockly 块形态一一对应（见下节）。
3. **块与字段**：块可包含**下拉框**、**数字输入**、**颜色选择框**等，参考 [Blockly Fields](https://raspberrypifoundation.github.io/blockly-samples/#fields)。
4. **多实例命名**：画布上同类型模块多于一个时，用 **A、B、C…** 区分（如 LED灯A、按键B）。
5. **拖入编程工作区**：所有块均可从工具箱拖到**编程工作区**中，进行组合与编程（与 [添加工具箱](https://developers.google.com/blockly/guides/get-started/toolbox?hl=zh-cn) 中「将块拖动到工作区」一致）。

---

## 二、与 Blockly 工具箱及块形态一致

依据 [Blockly 工具箱](https://developers.google.com/blockly/guides/get-started/toolbox?hl=zh-cn) 与 [块外观](https://developers.google.com/blockly/guides/design/appearance)：

- **工具箱**：用 **JSON** 定义工具箱内容（类别与块列表），传入工作区配置；块可从工具箱拖入工作区。
- **块形态**：四类对应 Blockly 的四种连接方式与用法：

| 分类 | Blockly 块形态 | 英文 | 说明 | 连接方式 |
|------|----------------|------|------|--------|
| **事件** | **帽子块** (Hat) | Hat block | 位于堆栈顶端，事件驱动入口 | 无上连接器；仅下方可接块 |
| **执行** | **堆叠块** (Stack) | Stack / Statement block | 顺序执行的语句 | 上、下堆叠连接（垂直） |
| **控制** | **C型块** (C-shaped) | C-shaped block | 内含语句槽，可嵌套块 | 内侧上方可接语句块，无内侧底连接（便于发现可堆叠多句） |
| **运算** | **数据块** (Value) | Value block | 返回值，作其他块的输入 | 横向拼图连接（与堆叠区分） |

- **事件使用帽子块**：如「当按下按键时」为帽子块，置于堆栈顶部。
- **执行使用堆叠块**：如「延迟」「LED灯A 显示」为堆叠块，上下依次连接。
- **控制使用 C 型块**：如「如果-那么」「重复 … 次」为 C 型块，内部可嵌套语句。
- **运算使用数据块**：如数字、比较、逻辑与/或，作为条件或参数嵌入其他块。

## 三、Blocks 分类与示例

| 分类 | 说明 | 示例 |
|------|------|------|
| **事件** | 帽子块，程序入口 | 当按下按键时 |
| **执行** | 堆叠块，具体动作 | 延迟（数字）秒；LED灯A 显示（颜色）；LED灯A 熄灭 |
| **控制** | C型块，逻辑与流程 | 如果-那么；重复（循环） |
| **运算** | 数据块，数学/逻辑/比较 | 数字；与/或；大于/小于 |

## 四、画布模块与 blocks 映射

### 4.1 输入模块：按键（input-button）

画布上有**按键**时，左侧出现：

| 分类 | 块 | 说明 |
|------|-----|------|
| **事件** | 当按下按键时 | 多按键时为「当按下按键A时」等 |

### 4.2 输出模块：LED 灯（output-led）

画布上有 **LED 灯**时，以下块出现在 **执行** 分类（多颗时为 LED灯A、LED灯B…）：

| 分类 | 块 | 字段 | 说明 |
|------|-----|------|------|
| **执行** | LED灯[实例] 显示 | 颜色选择框 | 点亮并设置颜色 |
| **执行** | LED灯[实例] 显示 … 持续 … 秒 | 颜色 + 数字（时间） | 显示指定颜色并持续若干秒 |
| **执行** | LED灯[实例] 亮度增加 … % | 数字（百分比） | 相对当前亮度增加百分比 |
| **执行** | LED灯[实例] 熄灭 | — | 关闭该 LED |

### 4.3 控制（逻辑，与画布模块无关）

**控制** 分类为各种逻辑块，始终可提供（或随主控存在时提供）：

| 分类 | 块 | 说明 |
|------|-----|------|
| **控制** | 如果-那么 | 条件分支 |
| **控制** | 如果-那么-否则 | 条件分支带否则 |
| **控制** | 重复 … 次 | 固定次数循环 |
| **控制** | 重复直到 … | 条件循环 |

### 4.4 主控 / 电源

主控、电源不直接对应块；主控存在时程序可运行。执行类中的「延迟」等与主控绑定。

## 六、与 Blockly 的对应关系

- **工具箱**：[添加工具箱](https://developers.google.com/blockly/guides/get-started/toolbox?hl=zh-cn) 建议用 **JSON** 定义（`kind: 'flyoutToolbox'` 或类别工具箱，`contents` 中 `kind: 'block'`, `type: '...'`）；ToyLab 左侧面板等价于工具箱，内容随画布动态生成，**所有块可拖到编程工作区**。
- **块形态**：事件=帽子块、执行=堆叠块、控制=C型块、运算=数据块；接入 Blockly 时可为每类设置对应 `output`/`previousStatement`/`nextStatement` 及 C 型内侧连接。
- **块外观**：[块外观](https://developers.google.com/blockly/guides/design/appearance) 建议：水平（数据）与垂直（堆叠）连接器视觉区分；C 型块内侧不留底连接，便于发现可堆叠多句。
- **代码生成**：可为每类块配置 Generator，导出 Python/微控制器代码。
