# 仿真画布设计规则

> 本文档约定仿真画布上电子模块的造型与交互逻辑，便于与真实电子模块、模块库保持一致，并统一后续扩展。

---

## 一、设计逻辑

### 1. 造型与真实电子模块一致

仿真画布中的模块**造型应与真实电子模块保持一致**，以便用户建立「画布即实物」的认知：

- **形状**：统一为**圆角方**（与实物模块外形一致），画布上节点尺寸统一为 **72×72px**（如 LED、按键、主控等）。
- **层次**：
  - **外壳**：当前统一使用与模块库一致的红色（电源/主色）作为模块块底色，保持通透糖果块效果（高光 + 同色阴影）。
  - **中心区域**：根据模块类型区分（如 LED 为白色灯罩、按键为红色凸起圆等），与实物上的主要可辨识区域对应。
- **右下角标识**：每个模块右下角有一枚**圆形 icon**，用于与模块库中的对应电子模块图标一致，便于识别类型；圆形底色按**模块分类**区分（见下条）。

---

### 2. 右下角 icon 圆形颜色按模块分类

画布上模块**右下角圆形 icon 的底色**按模块分类固定，与设计系统及页面中的模块分类色一致：

| 分类 | 色名 | 用途           | 页面中的颜色值（CSS） |
|------|------|----------------|------------------------|
| **输入** | 绿   | 输入类模块右下角圆 | `#10B981`（变量：`var(--color-input)`） |
| **输出** | 蓝   | 输出类模块右下角圆 | `#3B82F6`（变量：`var(--color-output)`） |
| **主控** | 紫   | 主控类模块右下角圆 | `#8B5CF6`（变量：`var(--color-mainctl)`） |
| **电源** | 红   | 电源类模块右下角圆 | `#FF7F8E`（变量：`var(--color-power)`） |

- 上述色值定义在 `prototype/layout.css` 的 `:root` 中，与模块库、设计系统（`docs/TOYLAB_DESIGN_SYSTEM.md`）中的模块分类色一致。
- icon 内的图形（如 LED 灯泡、按键、ESP32 等）与模块库中对应条目的图标一致，颜色一般为白色或浅色以保证在圆形底上可读。

---

## 二、页面中使用的颜色值汇总

以下为当前仿真画布与模块相关、在页面中实际使用的颜色值，便于实现与校对。

### 模块分类色（`:root` 变量，`layout.css`）

| 变量名 | 色值 | 说明 |
|--------|------|------|
| `--color-power`  | `#FF7F8E` | 电源 / 主色红；画布模块外壳、按键中间圆等 |
| `--color-mainctl`| `#8B5CF6` | 主控紫；主控类右下角 icon 圆 |
| `--color-input`  | `#10B981` | 输入绿；输入类右下角 icon 圆 |
| `--color-output` | `#3B82F6` | 输出蓝；输出类右下角 icon 圆 |

### 仿真画布中的具体用法

- **模块外壳（LED / 按键 / ESP32）**：背景 `var(--color-power)`，阴影 `rgba(255, 127, 142, 0.55)` / `rgba(255, 127, 142, 0.45)`。
- **按键中间凸起圆**：红色渐变，如 `#ff9aa5` → `#FF7F8E` → `#e86d7d`，与实物按键颜色一致。
- **右下角 icon 圆**：输入 `var(--color-input)`、输出 `var(--color-output)`、主控 `var(--color-mainctl)`；电源类若出现画布模块则用 `var(--color-power)`。

---

## 三、后续可补充规则

- 新增模块类型时，造型需符合「与真实电子模块一致」；右下角圆形 icon 按上表选用对应分类色。
- 若设计系统（`TOYLAB_DESIGN_SYSTEM.md`）中模块分类色调整，本规则中的色值及 `layout.css` 变量需同步更新。
