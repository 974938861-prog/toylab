# ToyLab 硬件编程网页 — 界面设计与布局说明

## 产品定位

- **网站名称**：ToyLab  
- **使用方式**：与电子模块连接 → 在网页上设计控制逻辑 → 烧录到硬件 → 模块按逻辑运行  
- **编程方式**：积木方式、Workflow、Python 三种

---

## 整体布局结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ① 顶部控制栏 (固定高度)                                                      │
├──────────────────────────────┬──────────────────────────────────────────────┤
│  ② 数字孪生画布               │  ④ 编程区                                     │
│  (可拖放模块、模拟输入/输出)   │  · 标头：积木 | Workflow | Python 切换        │
│                              │  · 编辑区                                     │
├──────────────────────────────┤                                              │
│  ③ 虚拟资源库                 │                                              │
│  · 已连接 / 电源 / 主控       │                                              │
│  · 输入 / 输出 模块           │                                              │
└──────────────────────────────┴──────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────────────────┐
│  ⑤ 底部控制台 E 区 (可折叠) — print / 系统日志 / 传感器实时数值               │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **左侧**：画布 + 资源库 上下分栏（可调高度）  
- **右侧**：编程区 占满高度  
- **底部**：控制台 固定或可折叠，高度可调  

---

## 1. 顶部控制栏

| 区域 | 内容 | 说明 |
|------|------|------|
| 左侧 | **文件** | 下拉或按钮：新建、打开、另存为、保存 |
| 中左 | **连接** | 按钮：「连接设备」/「断开」；点击后可选串口/设备 |
| 中 | **连接状态** | 图标 + 文案：未连接 / 已连接 (COMx) / 连接中… |
| 右侧 | **烧录** | 主按钮：「烧录」— 将当前逻辑烧录到已连接硬件 |

**可选**：标题 Logo「ToyLab」、主题/设置入口、帮助。

---

## 2. 左侧上方 — 数字孪生画布

| 项目 | 说明 |
|------|------|
| **作用** | 用户从下方虚拟资源库拖入模块，在此组成「虚拟硬件」并对其编程；支持模拟输入与模拟输出。 |
| **内容** | 空白画布 + 已放置的电子模块节点；模块间可连线表示电气/逻辑关系（可选）。 |
| **交互** | 拖拽放入、删除、选中、右键菜单；点击模块可聚焦到该模块相关编程。 |
| **模拟** | 输入模块（如按键）可点击模拟按下/松开；输出模块（如 LED）显示亮/灭等状态。 |

**布局**：占左侧上半部分，与下方资源库用可拖拽分隔条调节高度。

---

## 3. 左侧下方 — 虚拟资源库

| 分类 | 内容 | 说明 |
|------|------|------|
| **已连接的模块** | 列表（可为空） | 当前与真实设备连接并在画布上使用的模块；先做虚拟编程时可为空。 |
| **电源模块** | 21700 单节电池盒 | 一个可拖拽项，拖到画布表示供电。 |
| **主控模块** | （按后续产品定义） | 主控类模块，可先留位或与「主控」占位符。 |
| **输入模块** | 按键 | 可拖拽到画布，用于模拟按键输入。 |
| **输出模块** | LED 灯 | 可拖拽到画布，用于模拟 LED 输出。 |

**展示形式**：每类可折叠面板；每个模块为卡片或图标+名称，支持拖拽到上方画布。

---

## 4. 右侧 — 编程区

| 项目 | 说明 |
|------|------|
| **标头** | 三个 Tab：**积木方式** \| **Workflow** \| **Python**，切换后下方为对应编辑器。 |
| **积木方式** | 块式编程画布（如 Blockly），左侧块面板 + 中间搭积木区。 |
| **Workflow** | 节点+连线的流程图编辑（如流程图/状态机）。 |
| **Python** | 代码编辑器（语法高亮、简单自动补全），编辑 Python3 控制逻辑。 |

**布局**：右侧整列；Tab 下为当前模式的编辑区，可全屏该区域。

---

## 5. 底部 E 区 — 控制台

| 项目 | 说明 |
|------|------|
| **用途** | 显示 Python `print()` 输出、系统日志、传感器实时数值。 |
| **内容** | 多 Tab 或分段：**输出**（print）、**日志**（系统）、**传感器**（实时数值表格或曲线）。 |
| **交互** | 可折叠/展开；高度可拖拽调节；可清空、暂停刷新、导出日志。 |

**布局**：整页宽，在页面最底部，默认高度约 120–180px。

---

## 响应式与尺寸建议

- **最小宽度**：约 1280px，保证左侧（画布+资源库）与右侧编程区并排可用。  
- **左侧宽度**：约 320–400px，画布与资源库约 1:1 或 6:4 分配。  
- **右侧编程区**：占满剩余宽度。  
- **顶部**：约 48–56px；底部控制台默认约 120–180px。  

---

## 交互流程摘要

1. 用户在 **虚拟资源库** 中拖拽模块到 **数字孪生画布**。  
2. 在 **编程区** 选择积木 / Workflow / Python 之一编写逻辑（可与画布中模块绑定）。  
3. 在画布上 **模拟输入**（如点按键），观察 **模拟输出**（如 LED 状态）及 **控制台** 的 print/传感器。  
4. 通过 **顶部「连接」** 连接真实设备，**「烧录」** 将逻辑写入硬件。  

以上为 ToyLab 界面内容与布局设计说明，可作为实现与评审依据。
