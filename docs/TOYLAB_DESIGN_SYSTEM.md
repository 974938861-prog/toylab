# ToyLab 设计系统

> 视觉隐喻：电子模块像糖果一样放在玻璃上 —— Apple 风格 + 玻璃拟态（毛玻璃），主色红（#FF7F8E），清晰、专业。

---

## 一、整体风格

### 1.1 风格定位

- **Apple Design**：简洁、留白充足、层级清晰、圆角统一、无多余装饰。
- **玻璃拟态（Glassmorphism）**：半透明毛玻璃面板、背景模糊、轻边框与高光，让界面像「一块玻璃」，内容浮于其上。
- **糖果感模块**：画布与模块库中的「模块」视觉上像彩色糖果——饱满、光面、高饱和色块，与玻璃形成对比。

### 1.2 设计原则

- **主色（红 #FF7F8E）**仅用于**点睛**：关键按钮（运行、保存）、连线高亮、品牌 Logo，避免视觉过载。
- **整页背景**使用**浅白色** `#F2F4F7`，干净统一。
- **标头**使用**蓝色** `#4A00FF`，白字与白图标，关键按钮可继续用主色红点睛。
- **各区域（画布、模块库、编程区、控制台）**使用**强化毛玻璃**：浅蓝渐变底 + `blur(28px) saturate(160%)` + 内高光/外阴影，突出玻璃质感。
- 内容区以**浅色毛玻璃 + 白/浅灰**为主，保证可读性与专业感。
- 模块分类用固定色相（橙/紫/绿/蓝），与实物模块颜色可建立对应关系；电源用橙而非红，避免「危险」联想。
- 字体与间距偏 Apple：清晰、略疏、字重克制。

## 二、颜色体系

### 2.1 主色（红）

**色值**：`#FF7F8E`（珊瑚粉红）。

**使用范围（点睛，避免过载）**：
- **关键按钮**：运行、保存、写入设备等主要操作。
- **连线高亮**：画布或流程图中选中/激活的连线。
- **品牌 Logo**：ToyLab 字样或 Logo 图形。

不用于：大面积背景、普通链接、次要按钮。

| 用途     | 色值       | 说明     |
|----------|------------|----------|
| 主色     | `#FF7F8E`  | 珊瑚粉红 |
| 主色悬停 | `#e86d7d`  | 略深     |
| 主色浅   | `#FFE8EB`  | 浅红背景/高亮 |
| 主色玻璃 | `rgba(255,127,142,0.12)` | 毛玻璃着色用 |

### 2.2 整页背景与各区域毛玻璃

- **整页背景**：**浅白色** `#F2F4F7`，整页统一。
- **标头**：**蓝色** `#4A00FF` 实底，文字与图标为白色 `#ffffff` / `rgba(255,255,255,0.9)`，关键按钮可用主色红。
- **各区域毛玻璃**（画布、模块库、编程区、控制台，更透 + 磨砂感 + 厚边 + 大圆角）：
  - `background`: 半透明白泛淡紫渐变，透明度约 0.46–0.54；
  - `backdrop-filter: blur(40px) saturate(140%)`；
  - `-webkit-backdrop-filter` 同值；
  - 边框 `1px solid rgba(255,255,255,0.6)`；
  - `box-shadow`: 内顶/内底 2px 高光 + 内描边 + 外阴影，增强玻璃厚度感；
  - 圆角 `--radius-large: 24px`、`--radius-dock: 24px`，大圆角统一。
- **蓝底（标头）**：文字与标识使用**白色** `#ffffff` / `rgba(255,255,255,0.9)`。
- **白底 / 毛玻璃区**（画布、模块库、编程区、控制台）：文字与标识使用**灰色**，主文字 `#374151`、次要 `#6b7280`。
- **标头/控制台等按钮**：默认半透明白底 + 深蓝字，悬停更亮；**「写入设备」「保存」** 等关键按钮用主色 `#FF7F8E` 实底 + 白字，做点睛。

### 2.3 模块分类色（电源 / 主控 / 输入 / 输出）

| 分类 | 色名 | 色值       | 用途示例                 |
|------|------|------------|--------------------------|
| 电源 | 红   | `#FF7F8E`  | 电源分组、电源类模块块（与主色一致） |
| 主控 | 紫   | `#7C3AED`  | 主控分组、主控类模块块   |
| 输入 | 绿   | `#059669`  | 输入分组、输入类模块块   |
| 输出 | 蓝   | `#2563EB`  | 输出分组、输出类模块块   |

- 模块库分组：左侧色条或小圆点用上述色；列表项悬停可用对应浅色背景。
- 画布上的模块块：填充对应色，加高光/内阴影做「糖果」感。

### 2.4 中性色（背景、文字、边框）

| 用途       | 色值       |
|------------|------------|
| 页面背景   | `#F5F5F7`  |
| 面板背景   | `#FFFFFF` 或 毛玻璃白 |
| 次要背景   | `#FAFAFA`  |
| 主文字     | `#1D1D1F`  |
| 次要文字   | `#6E6E73`  |
| 占位/辅助  | `#8E8E93`  |
| 边框/分隔  | `#E5E5EA`  |
| 浅边框     | `#F2F2F7`  |

### 2.5 语义与状态

| 状态       | 色值       |
|------------|------------|
| 成功/已连接 | `#34C759` |
| 警告       | `#FF9F0A`  |
| 错误       | `#FF3B30`  |
| 禁用       | `#8E8E93`  |

## 三、玻璃拟态（毛玻璃）

### 3.1 使用场景

- **顶栏 / 底栏**：毛玻璃或实色，白字与半透明白按钮，关键按钮（如写入设备、保存）用主色实底点睛。
- **浮层、弹窗、部分面板**：半透明白 + 模糊，例如：
  - `background: rgba(255,255,255,0.72)`；
  - `backdrop-filter: blur(20px) saturate(180%)`；
  - `-webkit-backdrop-filter` 同值。
- **内容区面板**：可用轻毛玻璃（如 `rgba(255,255,255,0.85)` + `blur(12px)`）或纯白，与页面背景 `#F5F5F7` 形成层次。

### 3.2 玻璃边框与阴影

- 边框：`1px solid rgba(255,255,255,0.5)` 或 `rgba(0,0,0,0.06)`，细而轻。
- 阴影：`0 4px 24px rgba(0,0,0,0.06)` 一类，避免过重。

### 3.3 与「糖果」的层次

- 玻璃 = 背景层（顶栏、底栏、面板）。
- 糖果 = 前景层（画布上的模块块、模块库条目），实色 + 高光，不半透明，保证识别度。

## 四、比例与间距

### 4.1 圆角（Apple 感）

| 元素       | 圆角值   |
|------------|----------|
| 按钮       | 10px     |
| 输入框/卡片 | 12px   |
| 面板/大块  | 16px     |
| 模块块(糖果) | 12px   |

### 4.2 间距节奏

- 基础单位：**4px**，间距以 4 的倍数为主（4 / 8 / 12 / 16 / 20 / 24 / 32）。
- 面板内边距：**16px** 或 **20px**。
- 顶栏高度：**52px**（或 48px）；底栏可同或略小。
- 模块库列表项：高度约 **44px**，内部 padding 约 12px 16px。
- 分组之间：**12px** 或 **16px**。

### 4.3 模块（糖果）比例

- 画布上的模块块、模块库中的条目：视觉上偏**方形或略扁**，与实物方形模块一致。
- 图标或模块缩略图：方形或圆角方，比例约 1:1。

## 五、字体

### 5.1 字体族

- **中文界面**：`-apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`
- **英文/数字**：与上同，优先系统无衬线（Apple 设备上为 San Francisco）。
- **代码区（Python 等）**：`"SF Mono", "Menlo", "Monaco", "Consolas", monospace`

### 5.2 字号与字重

| 用途         | 字号  | 字重    |

| 页面标题     | 18px  | 700     |
| 面板标题     | 15px  | 600     |
| 正文         | 14px  | 400     |
| 辅助/提示    | 12px  | 400     |
| 按钮         | 13px  | 500     |
| 代码         | 13px  | 400     |

### 5.3 行高与颜色

- 正文行高：**1.5**。
- 主文字色：`#1D1D1F`；次要：`#6E6E73`；占位：`#8E8E93`。

## 六、组件约定（摘要）

- **顶栏 / 底栏**：毛玻璃或实色，白字；关键按钮（运行、保存、写入设备）用主色 `#FF7F8E` 实底 + 白字。
- **主按钮（运行、保存、写入设备）**：主色 `#FF7F8E` 实底 + 白字，圆角 10px，悬停略深。
- **次要按钮**：白半透明或浅灰，悬停略深。
- **连线高亮 / 品牌 Logo**：使用主色做点睛。
- **模块库分组**：左侧 4px 色条（橙/紫/绿/蓝），或小圆点。
- **画布上的模块块**：实色填充（对应分类色）+ 内阴影/高光做糖果感，圆角 12px。
- **毛玻璃面板**：半透明白或橙色 tint + `backdrop-filter: blur()`，细边框，轻阴影。

## 七、下拉菜单 / 弹出层规范

> 适用于所有右键菜单、下拉选项面板、颜色选择器等浮层元素，统一使用「浅红玻璃拟态」风格。

### 7.1 视觉风格

**红色玻璃拟态（Red Glassmorphism）**：以主色红为底色调的半透明毛玻璃，与页面整体白色玻璃拟态形成对比，强化操作层级感。

| 属性 | 规范值 | CSS 变量 |
|---|---|---|
| 背景色 | `rgba(220, 50, 70, 0.26)` | `var(--red-glass-bg)` |
| 背景模糊 | `blur(20px) saturate(160%)` | `var(--red-glass-blur)` |
| 边框 | `1px solid rgba(255, 180, 190, 0.45)` | `var(--red-glass-border)` |
| 阴影 | 红色外阴影 + 顶部白色内高光 | `var(--red-glass-shadow)` |
| 圆角 | `18px` | `var(--radius-popup)` |
| 文字颜色 | `#ffffff`（纯白） | — |

### 7.2 内部元素规范

| 子元素 | 规范 |
|---|---|
| 菜单项 / 按钮 | `color: #fff`，`font-weight: 500`，背景透明 |
| 菜单项悬停 | `background: rgba(255, 255, 255, 0.18)` |
| 输入框 | `color: #fff`，`background: rgba(255,255,255,0.18)`，`border: 1px solid rgba(255,255,255,0.35)` |
| 输入框聚焦 | `border-color: rgba(255,255,255,0.7)`，背景略亮 |
| 选项下拉（`<select>`） | 同输入框规范，白色箭头图标 |
| 标签文字 | `color: rgba(255,255,255,0.9)` |

### 7.3 实现方式（CSS）

```css
/* 统一变量定义（layout.css :root） */
--red-glass-bg:     rgba(220, 50, 70, 0.26);
--red-glass-blur:   blur(20px) saturate(160%);
--red-glass-border: 1px solid rgba(255, 180, 190, 0.45);
--red-glass-shadow: 0 8px 32px rgba(200, 40, 60, 0.28),
                    0 2px 8px rgba(0, 0, 0, 0.12),
                    inset 0 1px 0 rgba(255, 255, 255, 0.28);
--radius-popup: 18px;

/* 弹出层基础写法 */
.any-popup {
  background:          var(--red-glass-bg);
  backdrop-filter:     var(--red-glass-blur);
  -webkit-backdrop-filter: var(--red-glass-blur);
  border:              var(--red-glass-border);
  border-radius:       var(--radius-popup);
  box-shadow:          var(--red-glass-shadow);
  color:               #fff;
}
```

### 7.4 适用范围

- 画布节点右键菜单（`.canvas-context-menu`）
- 积木块右键菜单（同上 class 复用）
- 节点属性弹层（`.canvas-props-panel`）
- 颜色选择器弹出层（`.colour-picker-popup`）
- 今后所有新增的下拉菜单、Tooltip 弹层、Popover 均需遵循此规范

## 八、数字输入框规范

> 适用于所有可输入数值的字段，包括积木块内的参数输入、颜色选择器的通道数值等，统一使用「跑道圆（Pill）」造型。

### 8.1 视觉规格

| 属性 | 规范值 |
|---|---|
| 形状 | 跑道圆，`border-radius: 500px` |
| 背景 | 纯白 `#ffffff` |
| 文字颜色 | 灰色 `#6b7280` |
| 边框 | `1.5px solid rgba(0,0,0,0.1)` |
| 最小宽度 | `35px`（约等于高度的 2 倍，保证跑道形状） |
| 内边距 | `2px 8px`（水平留足半圆空间） |
| 字号 | `11px`，`font-weight: 500` |
| 对齐 | 居中 `text-align: center` |
| 聚焦态 | 边框加深 `rgba(0,0,0,0.28)` + 白色外发光 |
| 箭头 | 隐藏浏览器原生数字增减箭头 |

### 8.2 自适应宽度（积木块内）

积木块内的数字输入框宽度随输入内容自动伸长：

```
width = max(35px, 字符数 × 7px + 20px)
```

- 初始（空/单字符）固定 35px，保持跑道造型
- 内容增多时平滑扩展，不截断数字

### 8.3 适用范围

- 积木块调色板和工作区中所有 `field: 'number'` 参数框（`.block-field-number`）
- 颜色选择器弹出层中各通道数值输入框（`.cp-num-input`）
- 今后所有新增的数值输入场景均沿用此规范

### 8.4 CSS 实现参考

```css
.block-field-number,
.cp-num-input {
  background: #ffffff;
  color: #6b7280;
  border: 1.5px solid rgba(0, 0, 0, 0.1);
  border-radius: 500px;
  padding: 2px 8px;
  text-align: center;
  -moz-appearance: textfield;
}
.block-field-number::-webkit-inner-spin-button,
.block-field-number::-webkit-outer-spin-button,
.cp-num-input::-webkit-inner-spin-button,
.cp-num-input::-webkit-outer-spin-button { -webkit-appearance: none; }
```

## 九、积木块尺寸与换行规则

> 积木块（调色板 `.block-chip` 和工作区 `.workspace-block`）的宽度由内容驱动，**永远不换行**。

### 9.1 规则

| 规则 | 说明 |
|---|---|
| 随内容增宽 | `width: fit-content`，内容越长块越宽 |
| 禁止换行 | `white-space: nowrap`，文案、数字框、色块均在同一行内排列 |
| 去掉最大宽度限制 | 不设 `max-width`，确保长文案不被截断或折行 |
| 内部对齐 | `display: inline-flex; align-items: center`，文字与嵌入字段垂直居中 |

### 9.2 CSS 实现参考

```css
.block-chip,
.workspace-block {
  display: inline-flex;
  align-items: center;
  width: fit-content;
  white-space: nowrap;
  /* 不设 max-width */
}
.block-content {
  display: inline-flex;
  align-items: center;
  white-space: nowrap;
}
```

### 9.3 适用范围

- 编程区左侧调色板的所有积木块（`.blocks-palette-list .block-chip`）
- 编程区工作区的所有积木块（`.blocks-workspace .workspace-block`）
- 积木块内的所有子元素（文本、数字输入框、颜色色块）均跟随此规则不换行

## 十一、实施顺序建议

1. 先统一 **CSS 变量**（主色 `#FF7F8E`、模块四色 红/紫/绿/蓝、中性色、圆角、阴影）。
2. 关键按钮（运行、保存、写入设备）用主色实底点睛。
3. 内容区背景与面板应用**毛玻璃或浅色**。
4. 模块库与画布上的模块应用**分类色**（电源=橙）与**糖果感**。
5. 字体与间距按本节做一次通版微调。
