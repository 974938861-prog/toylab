## 无法打开网页时请检查

1. **toylab-service**：必须先启动。在 `toylab-service` 目录执行 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8001`，保持窗口不关。**编辑案例保存时若出现「Failed to fetch」**，多半是 service 未启动或未在 8001 端口运行，请先启动 toylab-service 再保存。
2. **toylab-web**：在 `toylab-web` 目录执行 `npm run dev`，等终端出现 “Ready” 后，按终端里显示的地址访问（如 http://localhost:3000 或 3001）。
3. **管理后台**：在 `toylab-admin/frontend` 目录执行 `npm run dev`，再访问 http://localhost:3003 。

（端口、账号或密码有变更时，本文件会一起更新。）

## 封面图存储（统一在 service）

- **存储位置**：`toylab-service/uploads/cases/`（由 service 的 `UPLOAD_DIR` 配置，默认 `./uploads`）。
- **访问方式**：通过 service 的 `/uploads` 路由。toylab-web 里封面用相对路径 `/uploads/cases/xxx.jpg`，由 Next 的 rewrite 代理到 service（默认 `http://localhost:8001`）。
- **上传方式**：使用管理端前端（http://localhost:3003）编辑案例并上传封面，或调用 service 的 `POST /api/admin/upload/cover`。

**若发现页/案例详情仍是占位图**：请先**启动 toylab-service**。若 service 使用其他端口，在 toylab-web 根目录建 `.env.local`，设置 `NEXT_PUBLIC_UPLOAD_ORIGIN=http://localhost:你的端口` 并重启 web，再硬刷新（Ctrl+F5）。

## 拆分后常见问题

- **图片不显示**：在 `toylab-service` 目录执行 `python run_fix_images.py`，为案例和商品写入占位图 URL；接口也会对空 `cover_url` 返回占位图。
- **中文变成问号**：在 `toylab-service` 目录执行 `python fix_utf8_data.py`，用 UTF-8 中文覆盖数据库中的名称/描述等字段；导入种子时请用 `mysql --default-character-set=utf8mb4`。
- **案例详情为空**：确保 toylab-service 已启动；第一个案例「Electro-Bean」在种子中有 BOM、步骤、资源与开发日志。
- **工作室**：页面内嵌了与原型一致的 studio 界面（`/prototype/index.html?view=studio`），UI 与交互以原型为准。

## 控制台报错说明

- **giveFreely.tsx / content_script.js**：来自浏览器扩展（如 Give Freely、沉浸式翻译等），**不是 ToyLab 代码**。可忽略；若干扰调试可在浏览器中暂时禁用相关扩展。
- **404 (Not Found)**：在开发者工具 → Network 里看是哪个请求返回 404。若是 `/api/cases`、`/api/auth/me` 等，说明 toylab-service 未启动或未在 8001 端口；先启动 toylab-service 再刷新页面。
- **保存时出现 Not Found**：管理端编辑案例保存时，会请求 `PUT /api/admin/cases/:id`、`PUT .../steps`、`PUT .../dev_logs`。若返回 404，请确认：① 已启动 **toylab-service**（端口 8001），不是 toylab-admin 的 Python 服务；② 若刚更新过代码，需**重启 toylab-service** 使新接口（如开发日志）生效。
- **开发日志新增一行不保存**：必须用 **toylab-service** 目录下的服务（端口 8001）。在项目里进入 `toylab-service` 目录，双击 `start.bat` 启动，或执行 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8001`。确保是该目录的最新代码并重启后再保存。
- **封面动态图上传报 404 (Not Found)**：请确认 8001 端口运行的是 **toylab-service**（本项目里的 `toylab-service` 目录），不是 toylab-admin 的 Python 服务或 backend。在 `toylab-service` 目录下执行 `start.bat` 或 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8001`。
- **封面动态图上传报 ERR_CONNECTION_RESET**：多为大文件经代理被重置，可尝试换小一点的 GIF/视频（建议 &lt;10MB）或重启 toylab-service 后再试。
