## 无法打开网页时请检查

1. **toylab-service**：必须先启动。在 `toylab-service` 目录执行 `python -m uvicorn app.main:app --host 127.0.0.1 --port 8001`，保持窗口不关。
2. **toylab-web**：在 `toylab-web` 目录执行 `npm run dev`，等终端出现 “Ready” 后，按终端里显示的地址访问（如 http://localhost:3000 或 3001）。
3. **管理后台**：在 `toylab-admin/frontend` 目录执行 `npm run dev`，再访问 http://localhost:3003 。

（端口、账号或密码有变更时，本文件会一起更新。）

## 封面图存储（统一在 service）

- **存储位置**：`toylab-service/uploads/cases/`（由 service 的 `UPLOAD_DIR` 配置，默认 `./uploads`）。
- **访问方式**：通过 service 的 `/uploads` 路由。toylab-web 里封面用相对路径 `/uploads/cases/xxx.jpg`，由 Next 的 rewrite 代理到 service（默认 `http://localhost:8001`）。
- **上传方式**：使用管理端前端（http://localhost:3003）编辑案例并上传封面，或调用 service 的 `POST /api/admin/upload/cover`。

**若发现页/案例详情仍是占位图**：请先**启动 toylab-service**。若 service 使用其他端口，在 toylab-web 根目录建 `.env.local`，设置 `NEXT_PUBLIC_UPLOAD_ORIGIN=http://localhost:你的端口` 并重启 web，再硬刷新（Ctrl+F5）。

## 拆分后常见问题

- **图片不显示**：在 `toylab-service` 目录执行 `python run_fix_images.py`，为案例和商品写入占位图 URL；接口也会对空 `cover_url` 返回占位图。
- **中文变成问号**：在 `toylab-service` 目录执行 `python fix_utf8_data.py`，用 UTF-8 中文覆盖数据库中的名称/描述等字段；导入种子时请用 `mysql --default-character-set=utf8mb4`。
- **案例详情为空**：确保 toylab-service 已启动；第一个案例「Electro-Bean」在种子中有 BOM、步骤、资源与开发日志。
- **工作室**：页面内嵌了与原型一致的 studio 界面（`/prototype/index.html?view=studio`），UI 与交互以原型为准。
